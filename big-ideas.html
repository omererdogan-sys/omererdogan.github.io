3.1. Variables and Assignments:

A variable is a placeholder that stores a value. You can assign a value to a variable (for example: `age ← 17`). Data types matter (number, text, boolean, etc.), and variable names are case‑sensitive.

3.2. Data Abstraction:

Using abstractions like lists (or arrays) and strings lets you store and manipulate more complex data. A list can hold multiple elements; each element has an index. Abstraction simplifies how you handle and organize data inside programs.

3.3. Mathematical Expressions & Sequencing:

Programs follow a sequence of instructions (statements) executed one after another. You can use mathematical expressions (with operators, order of operations, maybe MOD/remainder) to compute values or transform data in your program.

3.4. Strings:

Strings represent sequences of characters (words, sentences). You can combine (concatenate) strings, extract parts (substrings), and use them to build text output, messages, or process textual data in programs.

3.5. Boolean Expressions & Logic:

Boolean values (true/false) let the program make decisions. Using relational operators (like <, >, ==) and logical operators (AND, OR, NOT) you can compare values or conditions and control program flow based on those conditions.

3.6. Conditionals & Nested Conditionals:

If‑else statements let a program choose between different paths depending on a condition. Nested conditionals are conditionals inside other conditionals — useful when decisions are more complex or layered.

3.7. Iteration (Loops):

Loops let you repeat a block of code multiple times (e.g. “repeat until …” or “for each item in a list”). This is useful when you want to perform the same operation many times without writing the same lines over and over.

3.8. Lists and Collections:

Lists (or arrays) allow you to store many values under a single variable name. You can add, access, modify, or iterate through lists — helpful for handling groups of data like survey results, scores, names, etc.

3.9. Procedures (Functions) & Modularity:

Programs become easier to manage when you group related instructions into procedures (or functions). Instead of repeating code, you can write a procedure once and call it whenever needed. This helps reduce duplication and errors.

3.10. Developing Algorithms & Problem‑Solving Logic:

An algorithm is a step‑by‑step plan for solving a problem. In programming, you translate algorithms into code. Good design and clear logic are essential so your program works correctly and efficiently.

3.11. Algorithmic Efficiency & Complexity:

Some algorithms are faster or more efficient than others. Efficient algorithms use fewer resources (time, memory) and run faster — important when data or tasks are large. Understanding efficiency helps choose the best solution.

3.12. Advanced Concepts (Search, Randomness, Simulation):

Programs can use search algorithms (like binary search), randomness, and simulation to solve more complex problems. Simulations help model real‑world processes, while randomness and search help explore different scenarios or data sets.
